bibliothèque fstream

Jul 23, 20256 min read

  • Bibliothèque standard: <fstream>

La bibliothèque <fstream> fait partie de la bibliothèque standard C++.
Elle permet de manipuler des flux associés à des fichiers : lecture, écriture ou les deux, selon le type d’objet utilisé (std::ifstream, std::ofstream, std::fstream).

Inclusion

Pour utiliser les flux de fichiers :

Cela rend accessibles les objets et [[classs]s] suivantes :

  • std::ifstream : lecture depuis un fichier
  • std::ofstream : écriture dans un fichier
  • std::fstream : lecture et écriture

ainsi que les méthodes associées comme open(), close(), is_open(), etc.

Espace de nom

Comme toutes les bibliothèques de la STL, fstream appartient à l’espace de nom std.
Il faut donc écrire :

→ std::ifstream, std::ofstream, std::fstream

Ou importer l’espace de noms :

→ using namespace std

⚠️ L’import de using namespace std; reste déconseillé dans les fichiers d’en-tête.

std::ofstreamstd::ostreamÉcriture dans un fichier (écrase ou crée)
std::fstreamstd::iostreamLecture et écriture dans un même fichier

Les classes std::ifstream, std::ofstream, std::fstream sont des classe dérivées respectivement des flux de la bibliothèque iostream std::istream, std::ostream, std::iostream. Ces classes étendent les fonctionnalités de leurs classe de base respectives pour permettre la manipulation de fichiers. Pour autant, il n’est pas nécessaire d’inclure <iostream> si on veut se servir exclusivement des flux de fichiers.

Modes d’ouverture

Les flux de fstream acceptent plusieurs modificateurs de mode :

  • std::ios::in : lecture
  • std::ios::out : écriture
  • std::ios::app : ajout à la fin (append)
  • std::ios::trunc : tronque le fichier (supprime le contenu)
  • std::ios::binary : mode binaire (par défaut : texte)

Utilisation des opérateurs << et >>

  • << pour insertion dans une chaîne (comme std::cout)
  • >> pour extraction depuis une chaîne (comme std::cin)

Très utile pour formater des chaînes, convertir des nombres, ou parser du texte ligne par ligne.

Insertion dans un flux (<<)

L’opérateur << insère une représentation textuelle d’une valeur dans un flux de sortie (std::ostream, std::ofstream, std::stringstream, etc.).
Il n’ajoute aucun espace ni retour à la ligne automatique : c’est au programmeur de les insérer si nécessaire.
L’insertion est séquentielle et continue : tout ce qui est envoyé est écrit tel quel dans le flux.

Cet opérateur est surchargé pour les types standards (int, std::string, etc.), et peut l’être pour des classes personnalisées.

Lecture depuis un flux (>>)

L’opérateur >> extrait des données d’un flux d’entrée (std::istream, std::ifstream, std::istringstream, etc.) vers une variable.
Il ignore les caractères blancs (whitespace) en début de flux, puis lit jusqu’au prochain caractère blanc (espace, tabulation, saut de ligne).
La lecture s’interrompt dès qu’un séparateur est rencontré.

Pour lire une ligne complète, il faut utiliser std::getline(), car >> ne lit qu’un mot à la fois.

Positionnement du curseur

Les objets std::ifstream, std::ofstream et std::fstream permettent de manipuler le curseur de lecture ou d’écriture dans un fichier, grâce aux méthodes suivantes :

  • seekg(pos) : déplace le curseur de lecture (g pour get) à la position pos
  • seekp(pos) : déplace le curseur d’écriture (p pour put) à la position pos
  • tellg() : retourne la position actuelle du curseur de lecture
  • tellp() : retourne la position actuelle du curseur d’écriture

Ces fonctions sont utiles pour obtenir la taille d’un fichier, revenir au début, sauter des blocs de données, etc.

À propos de offset et des points d’ancrage

Les méthodes seekg() et seekp() acceptent un décalage (offset) exprimé en caractères (ou octets), ainsi qu’un point de départ (appelé ancrage) :

  • offset : entier indiquant le nombre de caractères à déplacer
  • std::ios::beg : depuis le début du fichier
  • std::ios::cur : depuis la position actuelle
  • std::ios::end : depuis la fin du fichier

À propos de std::ios::beg, cur, end

Les constantes std::ios::beg, std::ios::cur et std::ios::end sont des valeurs d’énumération appartenant à std::ios_base::seekdir.
Elles indiquent le point d’origine (ancrage) à partir duquel appliquer un décalage (offset) lors d’un repositionnement de curseur avec seekg() ou seekp() :

  • std::ios::beg → depuis le début du fichier
  • std::ios::cur → depuis la position actuelle
  • std::ios::end → depuis la fin du fichier

Ces constantes sont définies dans la bibliothèque <ios> (incluses automatiquement avec <fstream> ou <iostream>).

3. Prendre en compte les permissions du fichier

Si le fichier est en lecture seule, un std::ofstream échouera à l’ouverture.

4. Penser à fermer explicitement (close())

Même si le destructeur le fait automatiquement à la fin du bloc, il est bon usage de fermer manuellement si :

  • on veux s’assurer que les données sont bien écrites immédiatement

Remarques

  • Il est possible de tester si un fichier est ouvert avec is_open().
  • Toujours appeler close() à la fin pour libérer les ressources (ou laisser le destructeur le faire).
  • Par défaut, les fichiers sont traités en mode texte (les fins de ligne peuvent être interprétées selon le système d’exploitation).
  • fstream est compatible avec les manipulateur de flux comme std::setw, std::setprecision, etc.
